Infraestructura Automatizada en Google Cloud

Terraform · GitHub Actions · Ansible · Docker · Prometheus · Grafana · Cloudflare

Proyecto práctico de infraestructura real, desplegado y configurado de forma 100% automatizada.

Resumen del proyecto

Esta máquina virtual forma parte de un entorno GCP "live" gestionado con Terraform. El despliegue de la infraestructura, la configuración del sistema y los servicios que ves aquí están integrados en un flujo de trabajo DevOps completo:

  • Terraform para crear la infraestructura (VM, red, firewall, etc.).
  • GitHub Actions para ejecutar plan y apply de forma controlada.
  • Ansible para la configuración automática de la VM y despliegue de Docker.
  • Docker + docker-compose para levantar servicios como Grafana, Prometheus y esta web.
  • Prometheus + Blackbox + node_exporter para monitorizar tanto la VM como la propia página.
  • Cloudflare como capa de protección y publicación mediante dominio propio.

Arquitectura

El flujo general del proyecto es el siguiente:

  1. El repositorio de bootstrap define identidades, permisos y configuración inicial necesaria para desplegar la infraestructura.
  2. El repositorio de infra live define la infraestructura real: redes, VM, reglas de firewall, backends remotos, etc.
  3. Un pipeline de GitHub Actions ejecuta las fases de terraform fmt, validate, plan y apply, usando autenticación OIDC.
  4. Una vez creada la VM, Ansible la configura: instalación de Docker, despliegue de Prometheus, Grafana, Blackbox y esta web estática.
  5. Cloudflare publica el servicio por HTTPS y actúa como capa de protección y caché.

Monitorización y observabilidad

La infraestructura incluye un stack de observabilidad basado en Prometheus y Grafana:

  • Prometheus recolecta métricas de:
    • node_exporter → salud de la VM (CPU, memoria, disco, red).
    • blackbox-exporter → estado de esta página (HTTP 2xx, latencia, etc.).
    • Otros endpoints/servicios internos que se añadan en el futuro.
  • Grafana muestra dashboards personalizados para el entorno.

De esta forma se monitoriza tanto la infraestructura interna como la experiencia externa a través del dominio publicado mediante Cloudflare.
